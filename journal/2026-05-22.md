# 2026-05-22 — Sleep8 Sleep Metrics Ingestion

## What was built
- `ingestion/sleep.py` — vision LLM parses Sleep8 app screenshots, upserts into DuckDB
- `sleep_metrics` table in DuckDB: `resting_hr`, `hrv`, `breath_rate`, `time_to_fall_asleep_minutes`, `deep_sleep_minutes`
- `SLEEP_DROP_FOLDER` in `config/settings.py` pointing to `data/drop/sleep8/`
- Schema migration via `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` for idempotent re-runs

## Key decisions
- **Separate drop folder** (`data/drop/sleep8/`) rather than mixing Sleep8 and Garmin screenshots in the same folder. Keeps each ingestion script self-contained with no cross-source awareness.
- **Date from filename, not screenshot** — the vision LLM isn't asked for the date at all. The script looks for a `YYYY-MM-DD` pattern in the filename stem and skips the file with a warning if none is found. Simpler and more reliable than trusting the model to read dates from a mobile UI.
- **Nullable fields** — `time_to_fall_asleep_minutes` and `deep_sleep_minutes` are only populated if visible in the screenshot. The prompt returns `null` for missing metrics rather than guessing.
- **ON CONFLICT DO NOTHING** — re-running the script is safe; already-inserted dates are skipped.

## Problems faced
- Forgot to create the `data/drop/sleep8/` folder — the schema and script were in place but the folder didn't exist until explicitly created. The lazy `mkdir` in `run()` only creates the `processed/` subdirectory, not the parent.

## Concepts touched
- Vision LLM ingestion pattern (same as Garmin health screenshots)
- Idempotent schema migrations with `ALTER TABLE ... ADD COLUMN IF NOT EXISTS`
- Filename as a structured metadata source — avoids asking the model for information it might hallucinate
